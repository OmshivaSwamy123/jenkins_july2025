If you want the typical Git workflow in execution order, use these commands:

 1. Check current status

git status

2. Get the latest changes from the remote repository

git pull origin main

*(Replace `main` with your branch name if different.)*

3. Create a new branch (optional but recommended)

git checkout -b feature-branch

4. Make your code changes

5. See what changed

git status

6. Add files to staging

Add all files:

git add .

Or add a specific file:

git add filename.ext

7. Commit changes

git commit -m "Describe your changes"

8. Push branch to remote

First push:

git push -u origin feature-branch

Subsequent pushes:

git push

9. Switch back to main branch

git checkout main

10. Update main branch

git pull origin main

11. Merge your branch

git merge feature-branch

12. Push merged changes

git push origin main

First-time repository setup

If you are starting a new repository:


git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <repository-url>
git push -u origin main

Daily developer workflow (most common)

git pull origin main
git status
git add .
git commit -m "Your commit message"
git push origin main

Useful inspection commands

git log --oneline
git branch
git remote -v
git diff
git stash
git stash pop
